13.1 客户端预测（Client-Side Prediction）原理

13.1.1 预测的基本概念

客户端预测是解决网络延迟问题的核心技术。其基本思想是：客户端不等待服务器确认，立即执行玩家输入，模拟服务器的行为。

无预测 vs 有预测对比：

无预测（延迟明显）：
输入(t=0) ──► 发送到服务器 ──► 处理 ──► 返回结果 ──► 显示(t=200ms)

有预测（即时反馈）：
输入(t=0) ──► 立即执行并显示(t=0) ──► 等待服务器确认 ──► 调和(t=200ms)

预测的优势：

  • 即时反馈：玩家输入立即产生效果
  • 流畅体验：不受网络延迟影响
  • 可预测性：客户端和服务器使用相同物理逻辑

预测的劣势：

  • 实现复杂：需要处理预测错误
  • 计算开销：客户端需要模拟服务器逻辑
  • 安全问题：可能被作弊利用（需服务器验证）

13.2 服务器调和（Server Reconciliation）算法

13.2.1 调和机制

服务器调和是预测失败后的纠错机制。当服务器结果与预测不符时，客户端需要"回滚"到服务器状态，然后重新应用后续的输入。

using Godot;
using System;
using System.Collections.Generic;

namespace GameFramework.Network.Prediction
{
    /// <summary>
    /// 服务器调和管理器
    /// 处理预测结果与服务器结果的对比和调和
    /// </summary>
    public partial class ServerReconciliation : Node
    {
        // 调和配置
        [Export]
        public float MaxPositionError { get; set; } = 0.1f; // 最大位置误差

        [Export]
        public float MaxRotationError { get; set; } = 5f;   // 最大旋转误差（度）

        [Export]
        public float VelocityErrorThreshold { get; set; } = 0.5f;

        // 调和统计
        public int ReconciliationCount { get; private set; }
        public float AveragePositionError { get; private set; }
        public float AverageReconciliationTime { get; private set; }

        /// <summary>
        /// 检查是否需要调和
        /// </summary>
        public ReconciliationResult CheckReconciliation(
            PhysicsState predictedState,
            PhysicsState serverState,
            int inputSequence)
        {
            var result = new ReconciliationResult
            {
                SequenceNumber = inputSequence,
                PredictedState = predictedState,
                ServerState = serverState,
                NeedsReconciliation = false
            };

            // 计算位置误差
            float positionError = predictedState.Position.DistanceTo(serverState.Position);

            // 计算旋转误差
            float rotationError = predictedState.Rotation.AngleTo(serverState.Rotation);
            rotationError = Mathf.RadToDeg(rotationError);

            // 计算速度误差
            float velocityError = predictedState.Velocity.DistanceTo(serverState.Velocity);

            // 记录误差
            result.PositionError = positionError;
            result.RotationError = rotationError;
            result.VelocityError = velocityError;

            // 判断是否需要调和
            if (positionError > MaxPositionError ||
                rotationError > MaxRotationError ||
                velocityError > VelocityErrorThreshold)
            {
                result.NeedsReconciliation = true;
                ReconciliationCount++;

                // 更新统计
                AveragePositionError = (AveragePositionError * (ReconciliationCount - 1) + positionError)
                    / ReconciliationCount;

                GD.Print($"[调和] 序列号: {inputSequence}, " +
                        $"位置误差: {positionError:F3}m, " +
                        $"旋转误差: {rotationError:F2}°, " +
                        $"速度误差: {velocityError:F2}m/s");
            }

            return result;
        }

        /// <summary>
        /// 计算调和权重
        /// 根据误差大小决定调和强度
        /// </summary>
        public float CalculateReconciliationWeight(float error, float threshold)
        {
            if (error <= threshold * 0.5f)
                return 0f; // 小误差，不调和

            if (error >= threshold * 2f)
                return 1f; // 大误差，完全调和

            // 线性插值
            return (error - threshold * 0.5f) / (threshold * 1.5f);
        }

        /// <summary>
        /// 平滑调和
        /// 使用平滑插值避免画面抖动
        /// </summary>
        public PhysicsState SmoothReconcile(
            PhysicsState predicted,
            PhysicsState server,
            float deltaTime,
            float smoothingSpeed = 10f)
        {
            float t = smoothingSpeed * deltaTime;
            t = Mathf.Clamp(t, 0f, 1f);

            return new PhysicsState
            {
                Position = predicted.Position.Lerp(server.Position, t),
                Rotation = predicted.Rotation.Slerp(server.Rotation, t),
                Velocity = predicted.Velocity.Lerp(server.Velocity, t),
                Timestamp = server.Timestamp
            };
        }
    }

    /// <summary>
    /// 调和结果结构
    /// </summary>
    public struct ReconciliationResult
    {
        public int SequenceNumber;
        public PhysicsState PredictedState;
        public PhysicsState ServerState;
        public bool NeedsReconciliation;
        public float PositionError;
        public float RotationError;
        public float VelocityError;
    }
}

13.3 预测回滚（Prediction Rollback）完整实现

13.3.1 输入缓冲区设计

设计思路与原理

输入缓冲区是预测回滚系统的核心组件，存储玩家历史输入以便在需要时重新模拟。

缓冲区设计：

  1. 环形缓冲区：固定大小，旧输入被新输入覆盖
  2. 序列号：每个输入有唯一递增序列号
  3. 时间戳：记录输入发生的时间
  4. 输入数据：包含移动、射击、技能等所有操作

工作流程：

  1. 客户端记录本地输入到缓冲区
  2. 输入发送到服务器
  3. 服务器确认到达
  4. 若预测错误，从确认点重播输入

核心实现要点

  1. PlayerInput结构：存储单次输入的所有数据
  2. InputBuffer：环形缓冲区管理历史输入
  3. 序列号生成：单调递增防止重复
  4. 缓冲区溢出处理：最旧输入被覆盖

使用说明与最佳 practices

  • 适用场景：FPS、格斗游戏、竞速游戏等快节奏游戏
  • 注意事项：
    1. 缓冲区大小要足够存储最坏情况下的输入
    2. 输入要可序列化用于网络传输
    3. 重播输入要确定性和原始输入完全一致
  • 性能考虑：使用结构体而非类减少GC
  • 扩展建议：可添加输入压缩、输入校验等功能
using Godot;
using System;

namespace GameFramework.Network.Prediction
{
    /// <summary>
    /// 玩家输入结构
    /// 记录玩家的所有输入数据
    /// </summary>
    public struct PlayerInput : IEquatable<PlayerInput>
    {
        /// <summary>
        /// 输入序列号
        /// 单调递增，用于识别和排序输入
        /// </summary>
        public uint SequenceNumber;

        /// <summary>
        /// 输入时间戳（本地时间）
        /// </summary>
        public double Timestamp;

        /// <summary>
        /// 移动输入向量
        /// x: 左右, y: 0, z: 前后
        /// </summary>
        public Vector3 MovementInput;

        /// <summary>
        /// 观察/瞄准方向
        /// </summary>
        public Vector3 LookDirection;

        /// <summary>
        /// 跳跃按钮状态
        /// </summary>
        public bool JumpPressed;

        /// <summary>
        /// 下蹲按钮状态
        /// </summary>
        public bool CrouchPressed;

        /// <summary>
        /// 冲刺按钮状态
        /// </summary>
        public bool SprintPressed;

        /// <summary>
        /// 主要攻击按钮
        /// </summary>
        public bool AttackPressed;

        /// <summary>
        /// 技能按钮状态（位掩码）
        /// 每个位代表一个技能是否按下
        /// </summary>
        public uint AbilityMask;

        /// <summary>
        /// 鼠标Delta（用于精确瞄准）
        /// </summary>
        public Vector2 MouseDelta;

        /// <summary>
        /// 是否为空输入
        /// </summary>
        public bool IsEmpty => MovementInput.IsZeroApprox() &&
                              !JumpPressed && !CrouchPressed &&
                              !SprintPressed && !AttackPressed;

        /// <summary>
        /// 判断两个输入是否相等
        /// </summary>
        public bool Equals(PlayerInput other)
        {
            return SequenceNumber == other.SequenceNumber &&
                   MovementInput == other.MovementInput &&
                   JumpPressed == other.JumpPressed &&
                   CrouchPressed == other.CrouchPressed &&
                   SprintPressed == other.SprintPressed &&
                   AttackPressed == other.AttackPressed &&
                   AbilityMask == other.AbilityMask;
        }

        public override bool Equals(object obj)
        {
            return obj is PlayerInput other && Equals(other);
        }

        public override int GetHashCode()
        {
            return HashCode.Combine(SequenceNumber, MovementInput, JumpPressed,
                                   CrouchPressed, SprintPressed, AttackPressed, AbilityMask);
        }

        public override string ToString()
        {
            return $"[Seq:{SequenceNumber}] Move:{MovementInput} Jump:{JumpPressed} Crouch:{CrouchPressed}";
        }
    }

    /// <summary>
    /// 环形缓冲区
    /// 固定大小的循环队列，用于存储输入历史
    /// 时间复杂度：
    /// - Add: O(1)
    /// - Get: O(1)
    /// - Clear: O(1)
    /// 空间复杂度：O(Capacity)
    /// </summary>
    /// <typeparam name="T">存储的元素类型</typeparam>
    public class RingBuffer<T> where T : struct
    {
        // 存储数组
        private readonly T[] _buffer;

        // 容量
        public int Capacity { get; }

        // 当前数量
        public int Count { get; private set; }

        // 头尾指针
        private int _head; // 写入位置
        private int _tail; // 读取位置

        // 版本号（用于检测修改）
        private int _version;

        /// <summary>
        /// 构造函数
        /// </summary>
        /// <param name="capacity">缓冲区容量，建议为2的幂次以优化取模运算</param>
        public RingBuffer(int capacity)
        {
            if (capacity <= 0)
                throw new ArgumentException("容量必须大于0", nameof(capacity));

            // 将容量调整为2的幂次，优化性能
            Capacity = NextPowerOfTwo(capacity);
            _buffer = new T[Capacity];
            _head = 0;
            _tail = 0;
            Count = 0;
        }

        /// <summary>
        /// 获取下一个2的幂次
        /// </summary>
        private int NextPowerOfTwo(int n)
        {
            n--;
            n |= n >> 1;
            n |= n >> 2;
            n |= n >> 4;
            n |= n >> 8;
            n |= n >> 16;
            return n + 1;
        }

        /// <summary>
        /// 添加元素到缓冲区
        /// 如果缓冲区已满，覆盖最旧的元素
        /// </summary>
        public void Add(T item)
        {
            // 写入位置
            _buffer[_head] = item;

            // 移动头指针（使用位运算代替取模）
            _head = (_head + 1) & (Capacity - 1);

            // 如果缓冲区已满，移动尾指针
            if (Count == Capacity)
            {
                _tail = (_tail + 1) & (Capacity - 1);
            }
            else
            {
                Count++;
            }

            _version++;
        }

        /// <summary>
        /// 获取指定索引的元素
        /// 索引0表示最旧的元素
        /// </summary>
        public T Get(int index)
        {
            if (index < 0 || index >= Count)
                throw new ArgumentOutOfRangeException(nameof(index));

            // 计算实际位置
            int actualIndex = (_tail + index) & (Capacity - 1);
            return _buffer[actualIndex];
        }

        /// <summary>
        /// 获取最新添加的元素
        /// </summary>
        public T GetLast()
        {
            if (Count == 0)
                throw new InvalidOperationException("缓冲区为空");

            int index = (_head - 1) & (Capacity - 1);
            return _buffer[index];
        }

        /// <summary>
        /// 获取最旧的元素
        /// </summary>
        public T GetFirst()
        {
            if (Count == 0)
                throw new InvalidOperationException("缓冲区为空");

            return _buffer[_tail];
        }

        /// <summary>
        /// 查找指定序列号的元素
        /// 使用二分查找，时间复杂度O(log n)
        /// </summary>
        public bool TryGetBySequence(uint sequenceNumber, out T result)
        {
            result = default;

            // 遍历查找（针对序列号有序的情况可以优化为二分查找）
            for (int i = 0; i < Count; i++)
            {
                var item = Get(i);
                if (item is PlayerInput input && input.SequenceNumber == sequenceNumber)
                {
                    result = item;
                    return true;
                }
            }

            return false;
        }

        /// <summary>
        /// 移除指定数量最旧的元素
        /// </summary>
        public void RemoveOldest(int count)
        {
            if (count <= 0) return;
            if (count >= Count)
            {
                Clear();
                return;
            }

            _tail = (_tail + count) & (Capacity - 1);
            Count -= count;
            _version++;
        }

        /// <summary>
        /// 移除到指定序列号为止的所有输入
        /// </summary>
        public void RemoveUpToSequence(uint sequenceNumber)
        {
            int removeCount = 0;
            for (int i = 0; i < Count; i++)
            {
                var item = Get(i);
                if (item is PlayerInput input && input.SequenceNumber <= sequenceNumber)
                {
                    removeCount++;
                }
                else
                {
                    break;
                }
            }

            RemoveOldest(removeCount);
        }

        /// <summary>
        /// 获取从指定索引开始的所有元素
        /// </summary>
        public T[] GetRange(int startIndex, int count)
        {
            if (startIndex < 0 || startIndex >= Count)
                throw new ArgumentOutOfRangeException(nameof(startIndex));

            if (count < 0 || startIndex + count > Count)
                throw new ArgumentOutOfRangeException(nameof(count));

            T[] result = new T[count];
            for (int i = 0; i < count; i++)
            {
                result[i] = Get(startIndex + i);
            }

            return result;
        }

        /// <summary>
        /// 清空缓冲区
        /// </summary>
        public void Clear()
        {
            if (Count > 0)
            {
                Array.Clear(_buffer, 0, _buffer.Length);
                _head = 0;
                _tail = 0;
                Count = 0;
                _version++;
            }
        }

        /// <summary>
        /// 转换为数组
        /// </summary>
        public T[] ToArray()
        {
            T[] result = new T[Count];
            for (int i = 0; i < Count; i++)
            {
                result[i] = Get(i);
            }
            return result;
        }

        /// <summary>
        /// 是否包含指定序列号
        /// </summary>
        public bool ContainsSequence(uint sequenceNumber)
        {
            for (int i = 0; i < Count; i++)
            {
                var item = Get(i);
                if (item is PlayerInput input && input.SequenceNumber == sequenceNumber)
                {
                    return true;
                }
            }
            return false;
        }

        /// <summary>
        /// 获取最新序列号
        /// </summary>
        public uint GetLastSequenceNumber()
        {
            if (Count == 0)
                return 0;

            var last = GetLast();
            if (last is PlayerInput input)
            {
                return input.SequenceNumber;
            }
            return 0;
        }
    }
}

13.3.2 状态快照存储

设计思路与原理

状态快照存储实体的完整物理状态，用于在预测错误时恢复到正确状态并重播输入。

存储内容：

  1. 变换状态：位置、旋转、缩放
  2. 物理状态：速度、角速度、加速度
  3. 游戏状态：生命值、弹药、技能冷却
  4. 动画状态：当前动画、播放时间

快照管理：

  • 频率：以固定间隔（如66ms）保存快照
  • 数量：保存足够数量以应对最坏延迟
  • 存储方式：环形缓冲区，旧快照被覆盖
  • 压缩：只存储变化的状态减少内存

核心实现要点

  1. PhysicsState：单个实体的状态数据
  2. StateSnapshot：某一时刻所有实体的状态
  3. StateBuffer：状态快照缓冲区
  4. 状态恢复：从快照恢复实体状态

使用说明与最佳 practices

  • 适用场景：需要精确物理同步的竞技游戏
  • 注意事项：
    1. 快照要包含恢复所需的所有状态
    2. 状态恢复要完整，避免部分状态遗漏
    3. 考虑内存占用，及时清理旧快照
  • 性能考虑：使用结构体和内存池优化
  • 扩展建议：可添加增量快照、状态压缩等
using Godot;
using System;
using System.Collections.Generic;

namespace GameFramework.Network.Prediction
{
    /// <summary>
    /// 物理状态
    /// 记录实体的完整物理状态，用于回滚和重放
    /// </summary>
    public class PhysicsState
    {
        /// <summary>
        /// 状态时间戳
        /// </summary>
        public double Timestamp { get; set; }

        /// <summary>
        /// 对应的输入序列号
        /// 表示这个状态是执行了哪个输入后的结果
        /// </summary>
        public uint SequenceNumber { get; set; }

        /// <summary>
        /// 位置
        /// </summary>
        public Vector3 Position { get; set; }

        /// <summary>
        /// 旋转（四元数）
        /// </summary>
        public Quaternion Rotation { get; set; }

        /// <summary>
        /// 缩放
        /// </summary>
        public Vector3 Scale { get; set; } = Vector3.One;

        /// <summary>
        /// 线速度
        /// </summary>
        public Vector3 Velocity { get; set; }

        /// <summary>
        /// 角速度
        /// </summary>
        public Vector3 AngularVelocity { get; set; }

        /// <summary>
        /// 是否在地面
        /// </summary>
        public bool IsOnGround { get; set; }

        /// <summary>
        /// 地面法线
        /// </summary>
        public Vector3 GroundNormal { get; set; } = Vector3.Up;

        /// <summary>
        /// 额外状态数据（如血量、能量等）
        /// </summary>
        public Dictionary<string, object> ExtraData { get; set; }

        /// <summary>
        /// 状态是否有效
        /// </summary>
        public bool IsValid { get; set; } = true;

        public PhysicsState()
        {
            Position = Vector3.Zero;
            Rotation = Quaternion.Identity;
            Velocity = Vector3.Zero;
            AngularVelocity = Vector3.Zero;
            ExtraData = new Dictionary<string, object>();
        }

        /// <summary>
        /// 深拷贝
        /// </summary>
        public PhysicsState Clone()
        {
            var clone = new PhysicsState
            {
                Timestamp = Timestamp,
                SequenceNumber = SequenceNumber,
                Position = Position,
                Rotation = Rotation,
                Scale = Scale,
                Velocity = Velocity,
                AngularVelocity = AngularVelocity,
                IsOnGround = IsOnGround,
                GroundNormal = GroundNormal,
                IsValid = IsValid,
                ExtraData = new Dictionary<string, object>(ExtraData)
            };
            return clone;
        }

        /// <summary>
        /// 线性插值两个状态
        /// </summary>
        public static PhysicsState Lerp(PhysicsState from, PhysicsState to, float t)
        {
            return new PhysicsState
            {
                Timestamp = Mathf.Lerp(from.Timestamp, to.Timestamp, t),
                SequenceNumber = from.SequenceNumber, // 保持序列号
                Position = from.Position.Lerp(to.Position, t),
                Rotation = from.Rotation.Slerp(to.Rotation, t),
                Scale = from.Scale.Lerp(to.Scale, t),
                Velocity = from.Velocity.Lerp(to.Velocity, t),
                AngularVelocity = from.AngularVelocity.Lerp(to.AngularVelocity, t),
                IsOnGround = t > 0.5f ? to.IsOnGround : from.IsOnGround,
                GroundNormal = from.GroundNormal.Lerp(to.GroundNormal, t)
            };
        }

        /// <summary>
        /// 计算与另一个状态的差异
        /// </summary>
        public StateDelta GetDelta(PhysicsState other)
        {
            return new StateDelta
            {
                PositionDelta = other.Position - Position,
                RotationDelta = other.Rotation * Rotation.Inverse(),
                VelocityDelta = other.Velocity - Velocity,
                Timestamp = other.Timestamp
            };
        }

        /// <summary>
        /// 应用状态差异
        /// </summary>
        public void ApplyDelta(StateDelta delta)
        {
            Position += delta.PositionDelta;
            Rotation = delta.RotationDelta * Rotation;
            Velocity += delta.VelocityDelta;
        }

        public override string ToString()
        {
            return $"[Seq:{SequenceNumber}] Pos:{Position} Vel:{Velocity} Rot:{Rotation.GetEuler()}";
        }
    }

    /// <summary>
    /// 状态差异
    /// </summary>
    public struct StateDelta
    {
        public Vector3 PositionDelta;
        public Quaternion RotationDelta;
        public Vector3 VelocityDelta;
        public double Timestamp;
    }

    /// <summary>
    /// 状态快照管理器
    /// 存储和管理历史状态快照
    /// </summary>
    public class StateSnapshotManager
    {
        // 状态历史（按序列号索引）
        private Dictionary<uint, PhysicsState> _stateHistory;

        // 最大存储数量
        private readonly int _maxHistorySize;

        // 最新状态
        public PhysicsState LatestState { get; private set; }

        // 统计
        public int TotalSnapshotsStored { get; private set; }
        public int TotalSnapshotsDiscarded { get; private set; }

        public StateSnapshotManager(int maxHistorySize = 128)
        {
            _maxHistorySize = maxHistorySize;
            _stateHistory = new Dictionary<uint, PhysicsState>(maxHistorySize);
            LatestState = new PhysicsState();
        }

        /// <summary>
        /// 存储状态快照
        /// </summary>
        public void StoreSnapshot(PhysicsState state)
        {
            if (state == null || !state.IsValid)
            {
                GD.PushWarning("尝试存储无效状态");
                return;
            }

            // 存储到历史
            _stateHistory[state.SequenceNumber] = state.Clone();
            LatestState = state.Clone();
            TotalSnapshotsStored++;

            // 清理旧状态
            CleanupOldSnapshots();
        }

        /// <summary>
        /// 获取指定序列号的状态
        /// </summary>
        public PhysicsState GetSnapshot(uint sequenceNumber)
        {
            if (_stateHistory.TryGetValue(sequenceNumber, out var state))
            {
                return state.Clone();
            }
            return null;
        }

        /// <summary>
        /// 获取最近的快照
        /// </summary>
        public PhysicsState GetNearestSnapshot(uint sequenceNumber)
        {
            PhysicsState nearest = null;
            uint nearestDiff = uint.MaxValue;

            foreach (var kvp in _stateHistory)
            {
                uint diff = (uint)Math.Abs((int)(kvp.Key - sequenceNumber));
                if (diff < nearestDiff)
                {
                    nearestDiff = diff;
                    nearest = kvp.Value;
                }
            }

            return nearest?.Clone();
        }

        /// <summary>
        /// 获取序列号小于等于指定值的最新的快照
        /// </summary>
        public PhysicsState GetLastSnapshotBefore(uint sequenceNumber)
        {
            PhysicsState result = null;
            uint maxSeq = 0;

            foreach (var kvp in _stateHistory)
            {
                if (kvp.Key <= sequenceNumber && kvp.Key > maxSeq)
                {
                    maxSeq = kvp.Key;
                    result = kvp.Value;
                }
            }

            return result?.Clone();
        }

        /// <summary>
        /// 清理旧快照
        /// </summary>
        private void CleanupOldSnapshots()
        {
            while (_stateHistory.Count > _maxHistorySize)
            {
                // 找到最旧的序列号
                uint oldestSeq = uint.MaxValue;
                foreach (var seq in _stateHistory.Keys)
                {
                    if (seq < oldestSeq)
                        oldestSeq = seq;
                }

                _stateHistory.Remove(oldestSeq);
                TotalSnapshotsDiscarded++;
            }
        }

        /// <summary>
        /// 删除到指定序列号为止的所有快照
        /// </summary>
        public void ClearUpTo(uint sequenceNumber)
        {
            var toRemove = new List<uint>();
            foreach (var seq in _stateHistory.Keys)
            {
                if (seq <= sequenceNumber)
                {
                    toRemove.Add(seq);
                }
            }

            foreach (var seq in toRemove)
            {
                _stateHistory.Remove(seq);
            }
        }

        /// <summary>
        /// 清空所有历史
        /// </summary>
        public void Clear()
        {
            _stateHistory.Clear();
            LatestState = new PhysicsState();
        }

        /// <summary>
        /// 获取历史快照数量
        /// </summary>
        public int GetHistoryCount()
        {
            return _stateHistory.Count;
        }

        /// <summary>
        /// 获取状态历史的时间跨度
        /// </summary>
        public double GetHistoryTimeSpan()
        {
            if (_stateHistory.Count < 2)
                return 0;

            double minTime = double.MaxValue;
            double maxTime = double.MinValue;

            foreach (var state in _stateHistory.Values)
            {
                if (state.Timestamp < minTime)
                    minTime = state.Timestamp;
                if (state.Timestamp > maxTime)
                    maxTime = state.Timestamp;
            }

            return maxTime - minTime;
        }
    }

        /// <summary>
        /// 模拟输入效果
        /// 这是简化的物理模拟，应与服务器逻辑一致
        /// </summary>
        private void SimulateInput(ref PhysicsState state, PlayerInput input, double deltaTime)
        {
            // 基础移动速度
            float moveSpeed = input.SprintPressed ? 10f : 5f;

            // 计算目标速度
            Vector3 targetVelocity = input.MovementInput * moveSpeed;

            // 应用平滑
            state.Velocity = state.Velocity.Lerp(targetVelocity, (float)deltaTime * 10f);

            // 应用跳跃
            if (input.JumpPressed && state.IsOnGround)
            {
                state.Velocity += Vector3.Up * 10f;
                state.IsOnGround = false;
            }

            // 应用重力
            if (!state.IsOnGround)
            {
                state.Velocity += Vector3.Down * 20f * (float)deltaTime;
            }

            // 更新位置
            state.Position += state.Velocity * (float)deltaTime;

            // 简单的地面检测
            if (state.Position.Y < 0)
            {
                state.Position = new Vector3(state.Position.X, 0, state.Position.Z);
                state.Velocity = new Vector3(state.Velocity.X, 0, state.Velocity.Z);
                state.IsOnGround = true;
            }

            // 更新旋转（朝向移动方向）
            if (input.MovementInput.LengthSquared() > 0.001f)
            {
                Vector3 forward = input.MovementInput.Normalized();
                Basis targetBasis = new Basis(new Quaternion(Vector3.Up, Mathf.Atan2(forward.X, forward.Z)));
                state.Rotation = state.Rotation.Slerp(targetBasis.GetRotationQuaternion(), (float)deltaTime * 15f);
            }

            // 处理其他输入...
            if (input.CrouchPressed)
            {
                state.Scale = new Vector3(1, 0.5f, 1);
            }
            else
            {
                state.Scale = Vector3.One;
            }
        }

        // ============================================================
        // 核心功能：处理服务器确认
        // ============================================================

        /// <summary>
        /// 接收服务器状态确认
        /// 这是回滚机制的触发点
        ///
        /// 流程：
        /// 1. 接收服务器返回的权威状态
        /// 2. 对比预测结果与服务器结果
        /// 3. 如果不匹配，触发回滚
        /// 4. 从服务器状态重新模拟后续输入
        /// </summary>
        public void ReceiveServerConfirmation(uint sequenceNumber, PhysicsState serverState)
        {
            EmitSignal(SignalName.ServerStateReceived, sequenceNumber, serverState);

            // 更新最后确认序列号
            _lastConfirmedSequence = Math.Max(_lastConfirmedSequence, sequenceNumber);

            // 清理已确认的输入
            _inputBuffer.RemoveUpToSequence(sequenceNumber);

            // 获取对应的预测状态
            PhysicsState predictedState = _stateHistory.GetSnapshot(sequenceNumber);

            if (predictedState == null)
            {
                // 预测状态已丢失（可能缓冲区太小）
                GD.PushWarning($"[Prediction] 无法找到序列号 {sequenceNumber} 的预测状态");
                return;
            }

            // 检查是否需要调和
            if (!EnableReconciliation)
            {
                return; // 调和禁用，完全信任预测
            }

            // 计算误差
            float positionError = predictedState.Position.DistanceTo(serverState.Position);
            float rotationError = Mathf.RadToDeg(predictedState.Rotation.AngleTo(serverState.Rotation));

            // 检查是否超过误差阈值
            if (positionError > MaxPositionError || rotationError > MaxRotationError)
            {
                GD.Print($"[Prediction] 检测到偏差 Seq={sequenceNumber}, " +
                        $"位置误差={positionError:F3}m, 旋转误差={rotationError:F2}°");

                EmitSignal(SignalName.ReconciliationNeeded, positionError, rotationError);

                // 触发回滚
                PerformRollback(sequenceNumber, serverState);
            }
            else
            {
                // 误差在可接受范围内，微调即可
                if (positionError > MaxPositionError * 0.1f)
                {
                    // 轻微调和
                    SmoothCorrect(sequenceNumber, serverState);
                }
            }
        }

        // ============================================================
        // 核心功能：回滚与重放
        // ============================================================

        /// <summary>
        /// 执行回滚
        /// 核心算法：回滚到服务器状态，重新模拟所有后续输入
        ///
        /// 算法步骤：
        /// 1. 获取服务器确认的状态（回滚目标）
        /// 2. 获取从确认序列号+1到最新的所有输入
        /// 3. 清除这些输入对应的历史状态
        /// 4. 从服务器状态开始，重新执行每个输入
        /// 5. 存储新的预测状态
        /// 6. 应用最终结果到实体
        ///
        /// 复杂度：O(k)，k为需要重放的输入数量
        /// 最坏情况：所有输入都需要重放
        /// 平均情况：只有最近的几个输入需要重放（网络抖动）
        /// </summary>
        private void PerformRollback(uint confirmedSequence, PhysicsState serverState)
        {
            if (_isRollingBack)
            {
                GD.PushWarning("[Prediction] 检测到嵌套回滚，已跳过");
                return;
            }

            _isRollingBack = true;
            TotalRollbacks++;

            double startTime = Time.GetTicksMsec() / 1000.0;

            // 步骤1：获取需要重放的输入
            // 这些输入是在服务器确认之后本地产生的
            var inputsToReplay = GetInputsAfterSequence(confirmedSequence);

            GD.Print($"[Rollback] 开始回滚 Seq={confirmedSequence} -> {_nextSequenceNumber - 1}, " +
                    $"需要重放 {inputsToReplay.Length} 个输入");

            EmitSignal(SignalName.RollbackStarted, confirmedSequence, _nextSequenceNumber - 1, inputsToReplay.Length);

            // 步骤2：从服务器状态开始
            PhysicsState rollbackState = serverState.Clone();
            rollbackState.SequenceNumber = confirmedSequence;

            // 步骤3：清除从确认序列号之后的历史状态
            _stateHistory.ClearUpTo(confirmedSequence);

            // 步骤4：重新模拟每个输入
            int replayCount = 0;
            foreach (PlayerInput input in inputsToReplay)
            {
                // 模拟这个输入
                SimulateInput(ref rollbackState, input, GetPhysicsProcessDeltaTime());

                // 更新序列号和时间戳
                rollbackState.SequenceNumber = input.SequenceNumber;
                rollbackState.Timestamp = input.Timestamp;

                // 存储到历史
                _stateHistory.StoreSnapshot(rollbackState.Clone());

                replayCount++;
            }

            // 步骤5：更新当前预测状态
            _currentPredictedState = rollbackState.Clone();

            // 步骤6：应用最终状态到实体
            // 使用平滑过渡避免视觉跳跃
            ApplyStateToEntitySmooth(_currentPredictedState, SmoothRollbackSpeed);

            // 更新统计
            AverageRollbackInputs = (AverageRollbackInputs * (TotalRollbacks - 1) + replayCount) / TotalRollbacks;

            double endTime = Time.GetTicksMsec() / 1000.0;
            LastRollbackTime = (float)(endTime - startTime);

            _isRollingBack = false;

            EmitSignal(SignalName.RollbackCompleted, _currentPredictedState.SequenceNumber, _currentPredictedState);

            GD.Print($"[Rollback] 回滚完成，重放了 {replayCount} 个输入，" +
                    $"耗时 {LastRollbackTime * 1000:F2}ms, " +
                    $"最终位置 {_currentPredictedState.Position}");
        }

        /// <summary>
        /// 获取指定序列号之后的所有输入
        /// </summary>
        private PlayerInput[] GetInputsAfterSequence(uint sequenceNumber)
        {
            var result = new List<PlayerInput>();

            // 遍历输入缓冲区
            for (int i = 0; i < _inputBuffer.Count; i++)
            {
                PlayerInput input = _inputBuffer.Get(i);
                if (input.SequenceNumber > sequenceNumber)
                {
                    result.Add(input);
                }
            }

            // 按序列号排序
            result.Sort((a, b) => a.SequenceNumber.CompareTo(b.SequenceNumber));

            return result.ToArray();
        }

        /// <summary>
        /// 平滑修正
        /// 对于小误差，使用平滑插值而非立即回滚
        /// </summary>
        private void SmoothCorrect(uint sequenceNumber, PhysicsState serverState)
        {
            // 获取当前状态
            PhysicsState currentState = _stateHistory.LatestState;
            if (currentState == null) return;

            // 计算插值位置
            float t = GetPhysicsProcessDeltaTime() * SmoothRollbackSpeed;
            t = Mathf.Clamp(t, 0f, 1f);

            // 平滑过渡到服务器状态
            PhysicsState correctedState = PhysicsState.Lerp(currentState, serverState, t);
            correctedState.SequenceNumber = currentState.SequenceNumber;

            // 更新历史
            _stateHistory.StoreSnapshot(correctedState);
            _currentPredictedState = correctedState;

            // 应用
            ApplyStateToEntity(correctedState, true);
        }

        /// <summary>
        /// 将状态应用到实体
        /// </summary>
        private void ApplyStateToEntity(PhysicsState state, bool immediate)
        {
            if (_controlledEntity == null) return;

            if (immediate)
            {
                _controlledEntity.GlobalPosition = state.Position;
                _controlledEntity.GlobalRotation = state.Rotation.GetEuler();
                _controlledEntity.Velocity = state.Velocity;
                _controlledEntity.Scale = state.Scale;
            }
            else
            {
                // 使用插值平滑过渡
                ApplyStateToEntitySmooth(state, SmoothRollbackSpeed);
            }
        }

        /// <summary>
        /// 平滑应用状态到实体
        /// </summary>
        private void ApplyStateToEntitySmooth(PhysicsState state, float speed)
        {
            if (_controlledEntity == null) return;

            // 创建Tween进行平滑过渡
            var tween = CreateTween();
            tween.SetEase(Tween.EaseType.Out);
            tween.SetTrans(Tween.TransitionType.Quad);

            // 位置
            tween.Parallel().TweenProperty(
                _controlledEntity, "global_position", state.Position, 1f / speed);

            // 旋转
            tween.Parallel().TweenProperty(
                _controlledEntity, "global_rotation", state.Rotation.GetEuler(), 1f / speed);

            _controlledEntity.Velocity = state.Velocity;
        }

13.3.4 带预测回滚的角色控制器

设计思路与原理

预测角色控制器整合PredictionRollbackManager，为多人游戏提供流畅的移动体验。

工作流程：

  1. 本地预测：输入立即生效，玩家看到即时反馈
  2. 输入记录：所有输入记录到缓冲区并发送到服务器
  3. 服务器处理：服务器权威处理输入
  4. 结果对比：收到服务器结果后与预测对比
  5. 回滚修正：预测错误时回滚到正确状态并重播输入

控制器特性：

  • 平滑修正：预测错误时使用插值平滑修正
  • 最大修正：限制单次修正幅度防止瞬移
  • 容错处理：网络波动时的容错机制

核心实现要点

  1. PredictedCharacterController：继承CharacterBody3D
  2. 输入处理：_ProcessInput处理玩家输入
  3. 网络回调：OnServerStateReceived处理服务器结果
  4. 修正逻辑：ReconcilePosition执行位置修正

使用说明与最佳 practices

  • 适用场景：多人FPS、竞速游戏、格斗游戏
  • 注意事项：
    1. 预测逻辑要与服务器完全一致
    2. 修正要平滑避免突兀感
    3. 提供调试显示帮助调优
  • 性能考虑：优化物理模拟避免回滚时卡顿
  • 扩展建议：可添加武器预测、技能预测等

/*
正常情况（无回滚）：
- ProcessPlayerInput: O(1)
  - Add to buffer: O(1)
  - Simulate: O(1)
  - Store state: O(1)
- ReceiveServerConfirmation: O(1)
  - Get predicted state: O(1) (HashMap lookup)
  - Compare: O(1)

回滚情况：
- GetInputsAfterSequence: O(n)，n为缓冲区大小
- Simulate inputs: O(k * m)，k为需要重放的输入数，m为模拟复杂度
- Store states: O(k)
- 总体：O(n + k * m)

最坏情况：
- 所有输入都需要重放
- 复杂度：O(n * m)

平均情况：
- 网络稳定时k很小（1-3）
- 复杂度接近O(1)

空间复杂度：
- 输入缓冲区：O(n)，n为缓冲区容量
- 状态历史：O(s)，s为历史容量
- 总体：O(n + s)
*/

13.4.3 边界情况处理

using Godot;
using System;

namespace GameFramework.Network.Prediction
{
    /// <summary>
    /// 边界情况处理器
    /// 处理预测系统中的各种异常情况
    /// </summary>
    public class EdgeCaseHandler
    {
        // ============================================================
        // 1. 序列号回绕
        // ============================================================

        /// <summary>
        /// 处理序列号回绕
        /// uint溢出后会自动回绕到0，这是预期的行为
        /// 使用有符号数比较避免回绕问题
        /// </summary>
        public static bool IsSequenceNewer(uint newSeq, uint oldSeq)
        {
            // 转换为有符号数进行比较
            // 允许回绕，只要差值在合理范围内（半uint范围）
            const int HALF_UINT = int.MaxValue;
            int diff = (int)(newSeq - oldSeq);
            return diff > 0 && diff < HALF_UINT;
        }

        // ============================================================
        // 2. 极端延迟
        // ============================================================

        /// <summary>
        /// 处理极端延迟情况
        /// 当延迟超过缓冲区容量时的处理策略
        /// </summary>
        public static PhysicsState HandleExtremeLatency(
            PredictionRollbackManager manager,
            PhysicsState lastKnownState,
            float latency)
        {
            // 策略1：如果延迟过大，暂时禁用预测
            if (latency > 0.5f) // 500ms
            {
                GD.PushWarning($"极端延迟: {latency * 1000:F0}ms，切换为插值模式");
                // 切换到纯插值模式
                return lastKnownState;
            }

            // 策略2：平滑调和而非立即回滚
            // 避免大幅度跳跃
            return lastKnownState;
        }

        // ============================================================
        // 3. 输入缓冲区溢出
        // ============================================================

        /// <summary>
        /// 处理缓冲区溢出
        /// 当服务器确认太慢导致缓冲区填满时的处理
        /// </summary>
        public static void HandleBufferOverflow(RingBuffer<PlayerInput> buffer)
        {
            // 如果缓冲区满了但未收到确认，可能是网络问题
            if (buffer.Count >= buffer.Capacity * 0.9f)
            {
                GD.PushWarning("输入缓冲区即将溢出，可能存在网络问题");

                // 策略：保留最近的输入，丢弃最旧的
                int toRemove = buffer.Count / 4;
                buffer.RemoveOldest(toRemove);
            }
        }

        // ============================================================
        // 4. 服务器状态丢失
        // ============================================================

        /// <summary>
        /// 处理服务器状态丢失
        /// 当需要的状态不在历史记录中时
        /// </summary>
        public static PhysicsState HandleMissingState(
            StateSnapshotManager history,
            uint requiredSequence,
            PhysicsState currentState)
        {
            // 尝试获取最近的可用状态
            var nearest = history.GetNearestSnapshot(requiredSequence);

            if (nearest != null)
            {
                GD.Print($"使用最近的状态 Seq={nearest.SequenceNumber} 替代缺失的状态 Seq={requiredSequence}");
                return nearest;
            }

            // 如果连最近的状态都没有，使用当前状态并警告
            GD.PushError($"无法找到任何可用状态，使用当前状态进行调和");
            return currentState;
        }

        // ============================================================
        // 5. 预测死循环
        // ============================================================

        /// <summary>
        /// 防止预测死循环
        /// 当预测和服务器结果持续不匹配时
        /// </summary>
        public static bool DetectPredictionLoop(int rollbackCount, float timeWindow)
        {
            // 如果在短时间内回滚次数过多，可能存在逻辑不一致
            const int MaxRollbacksPerSecond = 30;

            if (rollbackCount > MaxRollbacksPerSecond * timeWindow)
            {
                GD.PushError("检测到预测死循环，客户端和服务器逻辑可能不一致");
                return true;
            }

            return false;
        }

        // ============================================================
        // 6. 实体传送
        // ============================================================

        /// <summary>
        /// 处理实体传送
        /// 当玩家被传送（如重生、传送门）时的特殊处理
        /// </summary>
        public static void HandleTeleport(
            PredictionRollbackManager manager,
            Vector3 newPosition)
        {
            GD.Print($"检测到传送，重置预测系统到位置: {newPosition}");

            // 重置预测管理器
            manager.Reset();

            // 创建新的初始状态
            var newState = new PhysicsState
            {
                Position = newPosition,
                Rotation = Quaternion.Identity,
                Velocity = Vector3.Zero,
                IsValid = true
            };

            // 立即应用
            manager.SetControlledEntity(null); // 需要重新设置
        }

        // ============================================================
        // 7. 网络断开与重连
        // ============================================================

        /// <summary>
        /// 处理网络断开
        /// </summary>
        public static void HandleDisconnection(PredictionRollbackManager manager)
        {
            GD.Print("网络断开，清空预测状态");
            manager.Reset();
        }

        /// <summary>
        /// 处理网络重连
        /// </summary>
        public static void HandleReconnection(PredictionRollbackManager manager)
        {
            GD.Print("网络重连，重新初始化预测系统");
            manager.Reset();
            // 可能需要重新同步服务器状态
        }

        // ============================================================
        // 8. 浮点精度问题
        // ============================================================

        /// <summary>
        /// 比较位置时考虑浮点精度
        /// </summary>
        public static bool PositionEquals(Vector3 a, Vector3 b, float epsilon = 0.001f)
        {
            return a.DistanceSquaredTo(b) < epsilon * epsilon;
        }

        /// <summary>
        /// 比较旋转时考虑浮点精度
        /// </summary>
        public static bool RotationEquals(Quaternion a, Quaternion b, float epsilon = 0.001f)
        {
            // 四元数有两种等价表示（q和-q）
            float dot = Math.Abs(a.Dot(b));
            return dot > 1f - epsilon;
        }
    }

13.5 核心概念深入讲解

13.5.1 预测的哲学基础

客户端预测的核心思想是"假装服务器会同意我的操作"。这基于一个假设：在大多数正常情况下，客户端的预测与服务器的结果是一致的。这个假设在合理设计的游戏中通常是成立的，因为客户端和服务器使用相同的物理模拟逻辑。

预测的心理学原理：人类对延迟的感知非常敏感。200ms的延迟会让操作感觉"粘滞"，而预测可以让操作感觉即时响应。即使偶尔需要回滚修正，只要回滚不频繁，玩家体验仍然是正向的。

确定性的重要性：预测系统要求客户端和服务器的行为是确定性的——给定相同的输入，必须产生相同的结果。这要求：

  • 使用固定时间步长（Fixed Time Step）
  • 避免浮点运算的精度差异（不同CPU架构可能产生微小差异）
  • 物理引擎配置完全一致
  • 不使用随机数（或种子同步）

13.5.2 回滚的代价与收益

回滚是预测系统的"纠错机制"，但它有代价：

回滚的开销：

  • CPU：需要重新模拟多个输入
  • 内存：需要存储历史状态
  • 网络：回滚期间可能产生额外的同步数据
  • 用户体验：视觉上的"跳跃"或修正

回滚的触发条件：

  • 预测位置与服务器位置偏差超过阈值
  • 发生了碰撞或触发（服务器权威）
  • 其他玩家的操作影响了当前玩家（被推、被击中等）
  • 网络延迟剧变导致时序混乱

优化回滚体验：

  • 使用平滑插值而非立即跳变
  • 只在偏差显著时触发（小误差可以容忍）
  • 优先修正位置和旋转，速度可以缓慢收敛

13.5.3 服务器调和的边界条件

服务器调和不是简单的"信任服务器"，而是在特定条件下触发：

应该调和的情况：

  • 位置偏差超过1个玩家身位
  • 速度方向明显错误（如向上飞但实际在地面）
  • 进入了不可能的区域（墙内、空中）
  • 资源数量不匹配（血量、弹药等）

可以容忍的偏差：

  • 位置偏差在10cm以内
  • 速度差异在10%以内
  • 旋转角度偏差小于5度
  • 短暂的视觉不一致

完全信任预测的场景：

  • 纯客户端效果（粒子、音效）
  • 不影响其他玩家的动画
  • UI反馈（准星、瞄准辅助）

13.6 实现细节深度分析

13.6.1 环形缓冲区的设计考量

环形缓冲区（RingBuffer）是预测系统的核心数据结构，设计时需要考虑：

容量计算： 缓冲区需要足够存储从输入发送到收到确认期间的所有输入。计算公式：

最小容量 = (最大RTT / 帧时间) + 安全余量
示例：RTT=200ms，60fps → 12帧 + 余量 = 64容量

内存布局优化： 使用2的幂次方容量可以利用位运算优化索引计算：

// 比取模运算快得多
index = (head + offset) & (capacity - 1);

序列号回绕处理： uint类型在超过4,294,967,295后会回绕到0。需要确保回绕时逻辑正确，通常通过比较序列号的差值而非绝对值来判断新旧。

多线程安全： 如果输入收集和网络回调在不同线程，需要使用锁或原子操作保护缓冲区。但在Godot单线程环境中，这个问题较少。

13.6.2 物理状态的快照与恢复

实现高效的状态快照需要精心的内存管理：

增量快照： 不必存储完整状态，只存储与上一帧的变化。例如，如果玩家站在原地，只需标记"无变化"而非发送完整Transform。

内存池： 频繁创建和销毁状态对象会产生GC压力。使用对象池预先分配状态对象，重复使用：

private PhysicsState GetPooledState()
{
    return _statePool.Count > 0 ? _statePool.Pop() : new PhysicsState();
}

状态压缩： 对于历史状态，可以压缩存储。例如，只保留位置（Vector3）和旋转（Quaternion），省略速度、加速度等可推导的值。

快照生命周期：

  • 输入被确认后，对应的快照可以丢弃
  • 维护一个"水印"指示最旧的有效快照
  • 定期清理过期快照，避免内存无限增长

13.6.3 平滑调和的算法选择

不同的调和算法产生不同的视觉效果：

指数平滑（Exponential Smoothing）：

// 越接近目标速度越慢
position = position + (target - position) * speed * delta;

优点：数学上平滑，无抖动。缺点：永远不能完全收敛到目标（渐进接近）。

线性插值（Linear Interpolation）：

position = Lerp(position, target, t);

优点：简单直观。缺点：需要计算合适的t值，太小则慢，太大则抖。

弹簧物理（Spring Physics）：

velocity += (target - position) * springStrength * delta;
velocity *= damping;
position += velocity * delta;

优点：物理上自然，有弹性效果。缺点：调参复杂。

临界阻尼（Critical Damping）： 弹簧物理的特殊情况，最快到达目标而不超调。适合大多数游戏场景。

推荐方案：

  • 位置：临界阻尼弹簧或指数平滑
  • 旋转：Slerp插值，使用较短的插值时间
  • 速度：直接设置（视觉影响小）

13.7 最佳实践指南

13.7.1 不同网络条件下的策略

低延迟（<50ms）环境：

  • 预测几乎总是正确
  • 可以放宽误差阈值（0.2m）
  • 减少回滚的平滑时间（快速收敛）
  • 甚至可以完全禁用回滚（完全信任预测）

中等延迟（50-150ms）环境：

  • 标准配置适用
  • 误差阈值：0.1m
  • 平滑时间：0.1-0.2秒
  • 定期清理历史快照（节省内存）

高延迟（>150ms）环境：

  • 增加缓冲区大小（更多历史记录）
  • 收紧误差阈值（0.05m）
  • 延长平滑时间（减少跳跃感）
  • 考虑增加插值延迟（牺牲响应换取平滑）

高抖动环境：

  • 使用抖动缓冲区平滑延迟波动
  • 指数移动平均估计延迟
  • 自适应阈值（延迟高时放宽阈值）
  • 优先保证流畅性而非精确性

13.7.2 预测系统的调试技巧

可视化工具：

  • 绘制预测轨迹（绿色）和服务器轨迹（红色）
  • 显示误差向量（当预测错误时）
  • 标记回滚触发点和幅度
  • 实时显示延迟和抖动统计

统计指标：

  • 回滚频率（每分钟的回滚次数）
  • 平均回滚输入数（每次回滚重放多少输入）
  • 最大误差（历史上最严重的偏差）
  • 回滚耗时（CPU开销）

调试命令：

13.7.3 常见陷阱与解决方案

陷阱1：浮点精度导致预测不一致

  • 现象：即使没有网络延迟，预测也会逐渐偏离
  • 原因：客户端和服务器使用不同的数学库优化
  • 解决：使用确定性的数学运算，或定期重新同步

陷阱2：物理引擎的非确定性

  • 现象：碰撞结果不一致
  • 原因：物理引擎使用多线程或随机算法
  • 解决：使用固定种子，禁用多线程物理，或让服务器权威处理碰撞

陷阱3：输入处理顺序错误

  • 现象：回滚后状态仍不匹配
  • 原因：输入未按正确顺序处理
  • 解决：确保输入按序列号严格排序

陷阱4：内存泄漏

  • 现象：运行时间越长内存占用越大
  • 原因：快照未正确清理
  • 解决：使用using语句，实现IDisposable，定期调用GC

陷阱5：嵌套回滚

  • 现象：回滚过程中又收到新的确认
  • 原因：网络延迟波动
  • 解决：使用_isRollingBack标志防止递归

13.7.4 性能优化建议

输入合并： 同一帧内的多个输入（如快速按键）可以合并为一个复合输入，减少需要重放的输入数量。

分层预测： 不是所有系统都需要预测。例如：

  • 位置和旋转：预测
  • 动画状态：插值
  • 粒子效果：仅客户端
  • UI元素：仅客户端

预测节流： 当检测到性能问题时，可以临时降低预测频率（如从60Hz降到30Hz），优先保证帧率。

异步回滚： 如果回滚计算量大（超过1ms），考虑在后台线程执行，避免阻塞渲染线程。注意线程同步问题。

本章小结：

  1. 客户端预测：玩家输入立即在本地执行，提供即时反馈
  2. 服务器调和：服务器验证预测结果，返回权威状态
  3. 预测回滚：当预测与服务器结果不符时，回滚并重新模拟
  4. 核心数据结构：
    • RingBuffer<PlayerInput>：高效的输入历史存储
    • PhysicsState：完整的状态快照
    • StateSnapshotManager：状态历史管理
  5. 复杂度：正常情况O(1)，回滚情况O(k)，k为需要重放的输入数

低延迟（<50ms）可适当放宽误差阈值
高延迟（>200ms）增加缓冲区大小，降低预测频率
