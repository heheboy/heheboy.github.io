第1章 Godot C# 核心架构

2026年4月1日 · 游戏开发

1.1 概述

Godot 引擎从 3.x 版本开始引入 C# 支持，在 4.x 版本中 C# 集成更加成熟和稳定。本章将深入介绍 Godot C# 的核心架构概念，帮助开发者理解如何在这个独特的节点系统中高效使用 C# 进行游戏开发。

1.2 为什么选择 Godot + C#

在游戏开发领域，开发者面临着众多技术选择。Unity、Unreal Engine、Godot 各有优势，而编程语言的选择同样重要。本节将分析为什么 Godot + C# 是一个值得考虑的技术组合。

1.2.1 Godot 的开源优势

Godot 采用 MIT 许可证完全开源，这意味着：

  • 零授权费用：无论游戏收入多少，都不需要支付引擎授权费或运行时费用
  • 源代码可访问：可以深入理解引擎工作原理，必要时修改引擎源码
  • 社区驱动：活跃的社区持续贡献代码、文档和教程
  • 长期可持续：即使官方开发放缓，社区可以接管维护

相比之下，Unity 在 2023 年宣布的运行时费用政策引发了开发者对商业引擎的担忧，而 Unreal Engine 虽然免费但收入超过阈值后需要支付 5% 版税。

1.2.2 Godot + C# vs Godot + GDScript

Godot 原生支持 GDScript，但选择 C# 有以下优势：

类型安全：C# 是强类型语言，编译时即可捕获类型错误，减少运行时 Bug。大型项目中，类型系统提供了更好的代码可维护性和 IDE 支持。

性能优势：C# 编译为 IL 后由 JIT 编译器优化，计算密集型任务性能明显优于 GDScript。对于需要复杂物理计算、AI 寻路或大规模数据处理的框架代码，C# 是更好的选择。

团队协作：C# 的静态类型和完整 IntelliSense 支持使团队协作更加顺畅。新成员可以更快地理解代码结构，IDE 提供了更好的重构和导航支持。

.NET 生态：可以直接使用 NuGet 生态系统的数千个库，从 JSON 序列化到 HTTP 客户端，从日志框架到测试工具，无需重复造轮子。

1.2.3 适合 Godot + C# 的项目类型

虽然 Godot + C# 适用于大多数游戏类型，但特别适合：

  • 中大型团队项目：需要强类型和良好架构支持
  • 复杂系统游戏：RPG、策略游戏等需要大量数据驱动逻辑
  • 性能敏感应用：需要充分利用 CPU 性能的计算密集型游戏
  • 已有 C# 技术栈的团队：可以复用现有的代码库和开发经验

1.4 Godot 节点树设计理念

Godot 的核心架构基于场景树（Scene Tree），所有游戏对象都是 Node 类的派生实例，通过父子关系组织成树形结构。理解节点生命周期是编写稳定游戏框架的基础。

1.4.1 为什么是树形结构

树形结构是游戏引擎中最自然的选择，原因如下：

层次化组织：游戏中的对象天然具有层次关系——玩家有装备，装备有属性，属性有数值。树形结构完美映射这种关系。

变换继承：子节点自动继承父节点的变换（位置、旋转、缩放）。这在实现如"玩家手持武器"这样的功能时非常自然——武器作为子节点，会跟随玩家移动。

生命周期管理：父节点负责子节点的生命周期。当父节点销毁时，子节点自动清理，避免内存泄漏。

作用域隔离：节点的信号、变量在其子树内有效，形成自然的封装边界。

1.4.2 与其他引擎的对比

Unity 的 GameObject：Unity 使用组件模式，GameObject 是容器，功能通过挂载 Component 实现。Godot 的节点既是容器也是功能实现者，更加统一。

Unreal 的 Actor：Unreal 的 Actor 类似 Godot 的 Node，但 Godot 的场景系统更加轻量，嵌套场景（Scene in Scene）的设计更加灵活。

自研引擎的实体系统：许多自研引擎使用扁平化的实体列表，缺乏层次关系。Godot 的树形结构提供了更好的组织性和可维护性。

1.4.3 节点树在游戏开发中的意义

理解节点树不仅是技术需求，更是设计思维：

场景即对象：在 Godot 中，场景（.tscn 文件）也是节点，可以被实例化为其他场景的子节点。这种"组合优于继承"的设计理念贯穿全书。

信号冒泡：事件和信号可以在节点树中传播，子节点可以通知父节点，形成自然的通信机制。

资源管理：节点的加载和卸载对应资源的加载和释放，树形结构使得资源依赖关系清晰可控。

1.5 C# 在 Godot 中的独特优势

Godot 4.x 对 C# 的支持达到了生产级别，.NET 6/8 的集成带来了独特优势。

1.5.1 现代 C# 特性支持

Godot 4.x 支持 C# 10/11 的特性：

可空引用类型：在项目中启用 <Nullable>enable</Nullable>，编译器可以帮助捕获空引用错误，提高代码健壮性。

记录类型（Record）：适合定义不可变的数据结构，如配置数据、事件数据等。

模式匹配：简化复杂的条件判断，使代码更加清晰。

异步编程：完整的 async/await 支持，配合 Godot 的 ToSignal 方法，可以编写清晰的异步代码。

1.5.2 调试体验

C# 在 Godot 中提供一流的调试体验：

Visual Studio / VS Code 集成：断点、单步执行、变量查看、调用堆栈等完整功能。

热重载（Hot Reload）：在调试模式下修改代码，可以立即看到效果，无需重新启动游戏。

异常处理：完整的异常堆栈信息，便于定位问题。

1.5.3 .NET 生态系统

使用 C# 意味着可以使用整个 .NET 生态系统：

System.Text.Json：高性能的 JSON 序列化，适合存档和网络通信。

Microsoft.Extensions.DependencyInjection：成熟的依赖注入框架，适合大型项目的架构设计。

开源库生态：从数据库 ORM 到机器学习，从网络库到数学计算，NuGet 提供了丰富的选择。

1.5.4 与 GDScript 的协作

在同一个项目中，C# 和 GDScript 可以无缝协作：

  • C# 可以调用 GDScript 的函数
  • GDScript 可以调用 C# 的方法
  • 信号系统跨语言工作
  • 共享相同的节点树和资源系统

这使得团队可以根据需求选择语言——框架核心用 C# 编写，快速原型或脚本用 GDScript 编写。

1.6 节点树生命周期

理解了为什么选择 Godot + C#，现在深入探讨节点树的核心机制——生命周期。这是编写稳定游戏框架的基石。

1.2.1 生命周期方法调用顺序

_Enumerate                  # 节点被识别并准备加入树
_EnterTree()               # 节点进入场景树（此时可在树中定位）
_Ready()                   # 节点及其子节点都准备完毕
_Process(delta)            # 每帧调用（渲染帧）
_PhysicsProcess(delta)     # 每物理帧调用（固定时间步长）
_ExitTree()                # 节点离开场景树

1.2.2 各生命周期方法详解

_EnterTree()

当节点被添加到场景树时调用，此时：

  • 节点可以通过 GetTree() 访问场景树
  • 可以通过 GetParent() 访问父节点
  • 子节点尚未完全初始化
// 文件路径: Scripts/Player/PlayerController.cs
// 功能说明: 玩家控制器，展示 _EnterTree 使用时机

using Godot;

namespace GameFramework.Player
{
    public partial class PlayerController : CharacterBody3D
    {
        // 在 _EnterTree 中注册到全局系统
        // 注意：此时还不能访问子节点
        public override void _EnterTree()
        {
            // 步骤1: 向全局事件系统注册自己
            // 这样其他系统可以在玩家创建时立即响应
            GameManager.Instance?.RegisterPlayer(this);

            // 注意：此时 _Ready 尚未调用，不要访问子节点
            // 错误示例：GetNode<AnimatedSprite3D>("Sprite") 可能返回 null
        }
    }
}

_Ready()

当节点及其所有子节点都完成 _EnterTree() 后调用。这是进行初始化的最佳时机：

  • 所有子节点都已可访问
  • 节点的导出属性已从编辑器或场景文件加载
  • 是获取子节点引用和设置初始状态的理想位置
// 文件路径: Scripts/Enemy/EnemyController.cs
// 功能说明: 敌人控制器，展示 _Ready 中的初始化

using Godot;

namespace GameFramework.Enemy
{
    public partial class EnemyController : CharacterBody3D
    {
        // 导出属性，可在编辑器中配置
        [Export] public float MoveSpeed { get; set; } = 5.0f;
        [Export] public int MaxHealth { get; set; } = 100;

        // 子节点引用
        private AnimatedSprite3D _sprite;
        private Area3D _attackRange;
        private HealthComponent _health;

        /// <summary>
        /// 初始化方法
        /// 执行顺序：
        /// 1. 获取子节点引用
        /// 2. 连接信号
        /// 3. 设置初始状态
        /// 4. 注册到管理器
        /// </summary>
        public override void _Ready()
        {
            // 步骤1: 获取子节点引用
            // 使用 @"NodePath" 语法避免转义问题
            _sprite = GetNode<AnimatedSprite3D>("Sprite");
            _attackRange = GetNode<Area3D>("AttackRange");
            _health = GetNode<HealthComponent>("HealthComponent");

            // 步骤2: 连接信号
            // 当玩家进入攻击范围时触发
            _attackRange.BodyEntered += OnPlayerEnteredRange;

            // 步骤3: 设置初始状态
            _health.CurrentHealth = MaxHealth;

            // 步骤4: 注册到敌人管理器
            EnemyManager.Instance.RegisterEnemy(this);
        }

        private void OnPlayerEnteredRange(Node3D body)
        {
            // 处理玩家进入攻击范围
        }
    }
}

_Process(delta)

每渲染帧调用一次，时间间隔不固定（取决于帧率）。适用于：

  • UI 更新
  • 视觉效果
  • 输入检测（非物理相关）
  • 动画控制
// 文件路径: Scripts/UI/FloatingDamageText.cs
// 功能说明: 浮动伤害数字，展示 _Process 使用

using Godot;

namespace GameFramework.UI
{
    public partial class FloatingDamageText : Label
    {
        [Export] public float FloatSpeed { get; set; } = 50.0f;
        [Export] public float FadeSpeed { get; set; } = 1.0f;

        private float _lifetime = 2.0f;

        /// <summary>
        /// 每帧更新位置和透明度
        /// 使用 delta 确保动画速度不受帧率影响
        /// </summary>
        public override void _Process(double delta)
        {
            // 步骤1: 向上浮动
            // delta 是上一帧到当前帧的时间（秒）
            Position += new Vector2(0, -FloatSpeed * (float)delta);

            // 步骤2: 逐渐淡出
            Modulate = new Color(Modulate.R, Modulate.G, Modulate.B,
                Modulate.A - FadeSpeed * (float)delta);

            // 步骤3: 生命周期检查
            _lifetime -= (float)delta;
            if (_lifetime <= 0 || Modulate.A <= 0)
            {
                // 从场景中移除
                QueueFree();
            }
        }
    }
}

_PhysicsProcess(delta)

每物理帧调用，使用固定时间步长（默认 60fps）。适用于：

  • 物理移动和碰撞
  • 角色控制器
  • 需要稳定时间步长的逻辑
// 文件路径: Scripts/Player/PhysicsBasedMovement.cs
// 功能说明: 基于物理的角色移动

using Godot;

namespace GameFramework.Player
{
    public partial class PhysicsBasedMovement : CharacterBody3D
    {
        [Export] public float Speed { get; set; } = 10.0f;
        [Export] public float JumpVelocity { get; set; } = 15.0f;
        [Export] public float Gravity { get; set; } = 50.0f;

        /// <summary>
        /// 物理更新
        /// 重要：所有物理相关计算都应在此进行，确保一致性
        /// </summary>
        public override void _PhysicsProcess(double delta)
        {
            Vector3 velocity = Velocity;

            // 步骤1: 处理重力
            if (!IsOnFloor())
            {
                // 注意：即使使用 _PhysicsProcess，仍应使用 delta
                // 以防物理帧率被修改
                velocity.Y -= Gravity * (float)delta;
            }

            // 步骤2: 处理跳跃
            if (Input.IsActionJustPressed("jump") && IsOnFloor())
            {
                velocity.Y = JumpVelocity;
            }

            // 步骤3: 处理水平移动
            Vector2 inputDir = Input.GetVector("move_left", "move_right",
                "move_forward", "move_back");
            Vector3 direction = new Vector3(inputDir.X, 0, inputDir.Y).Normalized();

            if (direction != Vector3.Zero)
            {
                velocity.X = direction.X * Speed;
                velocity.Z = direction.Z * Speed;
            }
            else
            {
                // 减速到停止
                velocity.X = Mathf.MoveToward(velocity.X, 0, Speed);
                velocity.Z = Mathf.MoveToward(velocity.Z, 0, Speed);
            }

            // 步骤4: 应用速度并移动
            Velocity = velocity;
            MoveAndSlide();  // Godot 4.x 的滑动碰撞处理
        }
    }
}

_ExitTree()

当节点从场景树中移除时调用，用于清理工作：

  • 取消信号连接
  • 注销全局注册
  • 释放资源
// 文件路径: Scripts/Systems/AudioManager.cs
// 功能说明: 音频管理器，展示 _ExitTree 清理

using Godot;
using System.Collections.Generic;

namespace GameFramework.Systems
{
    public partial class AudioManager : Node
    {
        // 正在播放的音频池
        private List<AudioStreamPlayer> _activePlayers = new();

        public void PlaySound(AudioStream sound, float volume = 1.0f)
        {
            var player = new AudioStreamPlayer();
            AddChild(player);
            player.Stream = sound;
            player.VolumeDb = Mathf.LinearToDb(volume);
            player.Play();
            _activePlayers.Add(player);

            // 播放完成后自动清理
            player.Finished += () => OnSoundFinished(player);
        }

        private void OnSoundFinished(AudioStreamPlayer player)
        {
            _activePlayers.Remove(player);
            player.QueueFree();
        }

        /// <summary>
        /// 清理工作
        /// 重要：必须手动清理，否则可能造成内存泄漏
        /// </summary>
        public override void _ExitTree()
        {
            // 步骤1: 停止所有正在播放的音频
            foreach (var player in _activePlayers)
            {
                if (IsInstanceValid(player))
                {
                    player.Stop();
                    player.QueueFree();
                }
            }
            _activePlayers.Clear();

            // 步骤2: 注销全局引用
            if (GameManager.Instance != null)
            {
                GameManager.Instance.AudioManager = null;
            }

            GD.Print("[音频管理器] 已清理并退出");
        }
    }
}

理解 C# 和 GDScript 的差异有助于在 Godot 中做出正确的技术选择。

1.3.1 语法与类型系统对比

特性C#GDScript

1.3.2 API 使用差异

// ==================== C# 版本 ====================

// 文件路径: Scripts/Examples/CSharpExample.cs

using Godot;
using System.Collections.Generic;
using System.Linq;

namespace GameFramework.Examples
{
    public partial class CSharpExample : Node
    {
        // 导出属性使用 C# 属性语法
        [Export] public int MaxCount { get; set; } = 10;
        [Export] public float MoveSpeed { get; set; } = 5.5f;
        [Export] public string PlayerName { get; set; } = "Player";

        // 导出枚举
        public enum GameDifficulty { Easy, Normal, Hard }
        [Export] public GameDifficulty Difficulty { get; set; } = GameDifficulty.Normal;

        // 导出资源
        [Export] public PackedScene EnemyPrefab { get; set; }
        [Export] public Texture2D IconTexture { get; set; }

        public override void _Ready()
        {
            // 获取节点 - C# 使用泛型方法
            var player = GetNode<CharacterBody3D>("Player");
            var ui = GetNode<CanvasLayer>("UI");

            // 获取子节点组
            var enemies = GetTree().GetNodesInGroup("enemy")
                .Cast<EnemyController>()  // 类型转换
                .ToList();

            // 信号连接 - C# 使用 += 语法
            player.TreeExiting += () => GD.Print("玩家离开场景");

            // 延迟调用 - 使用 async/await
            _ = DelayedAction();
        }

        private async System.Threading.Tasks.Task DelayedAction()
        {
            await ToSignal(GetTree().CreateTimer(2.0), "timeout");
            GD.Print("2秒后执行");
        }

        // 信号声明 - C# 使用 [Signal] 特性
        [Signal]
        public delegate void HealthChangedEventHandler(int newHealth, int maxHealth);

        [Signal]
        public delegate void PlayerDiedEventHandler();

        public void TakeDamage(int damage)
        {
            // 发射信号
            EmitSignal(SignalName.HealthChanged, 80, 100);
        }
    }
}

// 文件路径: Scripts/Examples/CollectionComparison.cs

using Godot;
using Godot.Collections;  // Godot 集合命名空间
using System.Collections.Generic;

namespace GameFramework.Examples
{
    public partial class CollectionComparison : Node
    {
        public override void _Ready()
        {
            // ========== 数组 ==========

            // C# 方式：使用 System 数组或 List
            List<int> scores = new List<int> { 100, 85, 92 };
            int[] fixedScores = new int[] { 100, 85, 92 };

            // Godot 方式：用于与 GDScript 交互
            Godot.Collections.Array godotArray = new Godot.Collections.Array();
            godotArray.Add(100);
            godotArray.Add(85);

            // ========== 字典 ==========

            // C# 方式：强类型字典
            Dictionary<string, int> playerScores = new Dictionary<string, int>
            {
                ["Player1"] = 1000,
                ["Player2"] = 850
            };

            // 访问：强类型，编译时检查
            if (playerScores.TryGetValue("Player1", out int score))
            {
                GD.Print($"Player1 分数: {score}");
            }

            // Godot 方式：用于导出到编辑器或与 GDScript 交互
            Godot.Collections.Dictionary godotDict = new Godot.Collections.Dictionary();
            godotDict["health"] = 100;
            godotDict["mana"] = 50;

            // ========== 遍历 ==========

            // C# LINQ - 强大的查询能力
            var highScores = playerScores
                .Where(kvp => kvp.Value > 900)
                .Select(kvp => kvp.Key)
                .ToList();

            // C# foreach
            foreach (var kvp in playerScores)
            {
                GD.Print($"{kvp.Key}: {kvp.Value}");
            }
        }
    }
}
